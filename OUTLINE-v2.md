## 〇、核心需求与总叙事（修订版）

**一句话**：AI 的计量单位正在从 token 换成"任务"——但这把新尺子现在量出来的数字里，有多少是真需求，多少是浪费和补贴？

**为什么需要它**：所有公开讨论都卡在 token 口径上（"AI 通缩"vs"算力紧缺"同时为真）；换尺子能解开，但新尺子自身有两个嫌疑犯：**浪费率**（30-72% 的 token 是低效消耗，可被工程压缩）和**补贴**（$200 订阅交付 $8-14k API 等效）。报告的任务是裁决这三件事。

**判断强度三级标签**：已验证 / 有证据有争议 / 假说待裁决——标题强度永远不超过证据强度。

## 链条（六个扣，修订）

```
剪刀差（钩子）→ 两条价格曲线（旧尺子）→ 新尺子与两个嫌疑犯（谱系+浪费率+补贴）
→ 三个裁决（H2/H3/H4）→ 边界（瓶颈接力）→ 谁付账（资产负债表）→ 分支开关（触发器 v2）
```

## 引子 · 剪刀差（修订：弃用"矛盾"框架）

- **要读者相信**：两个各自为真的事实，单层口径拼不起来。
- **论证**：GPT 旗舰 2023→2025 中降 24 倍（时间戳化）vs Uber 四个月烧完全年预算（年化约 3 倍超支，不用"10 倍"）。结尾句改为："**两层事实各对一半——只用一把尺子量，量出来的全是错觉。**"
- 贯穿案例（读者审查建议）："2024 年让 AI 总结一篇文档 ≈ 5K token、约 ¥0.1；2026 年让 AI 写完一份竞品分析 ≈ 10⁷ token、约 ¥300——同一个'帮我干活'，剂量变了 2,000 倍。"（标为示意算例）
- 图：双轨迹（价格下行线 vs 企业支出上行线）。

## §1 · 旧尺子：两条价格曲线（修订：双曲线框架）

- **要读者相信**："同等性能在通缩"和"最强能力在涨价"是两条曲线，不是一条。
- **论证**：V 形轨迹（GPT-5 $1.25 谷底 → GPT-5.5 Pro $30 新设超档、Fable $10/$50）+ GPT-5.6 Luna $1/$6（反例入列）+ **中国旗舰分化**（K3=K2 的 5 倍 ↑ / Qwen ~2x ↑ / MiniMax M3 半价 ↓ / DeepSeek 75 折守地板）。全部官方定价页。
- 强度：有证据有争议（无提价后留存数据前，不给"已验证"）。
- 过渡：为什么有人愿意为前沿付数十倍价？因为买的不是 token。

## §2 · 新尺子与两个嫌疑犯（重写：谱系 + 浪费率 + 补贴）

- **要读者相信**：token 是中间品（原油），任务是产品（里程）；任务强度横跨 10³→10⁹——但这把尺子读数里掺着水分。
- **论证**：
  1. **谱系图（地基主图，log 轴）**：对话 <1K / 单文档 5K / 多文档 10⁵ / agentic coding 10⁶-10⁷（斯坦福：1,200-3,500×，标"单一研究、特定 benchmark"）/ 应用构建 10⁸-10⁹（标"单源待复核"）。每档报区间+对数中点（run 间方差最高 30×）。
  2. **官方换算锚（降格表述）**：Salesforce 7,526 token/AWU——"首个公开 token→任务换算锚（单一厂商 CRM 微任务口径，仅作量级参考）"；附边际漂移与口径 caveat。"**统一尺子还不存在——谁造出它，正是 H3 的战争。**"
  3. **嫌疑犯一：浪费率**。双列呈现"有效强度/浪费系数"：《财经》30-60%、实测 72%（200+ 文件代码库）、Entelligence 每 $1 仅 18¢ 触达价值；反面：caching -90%、routing、批价 50% 正在商品化压缩。**任务通胀 = 有效强度 × (1+浪费率)——压缩浪费的技术栈就是空头的武器。**
  4. **嫌疑犯二：补贴**。SemiAnalysis $200 订阅 = $8-14k API 等效（标"API 标价口径，非成本口径"）。
- 过渡：带着两个嫌疑犯，裁决三个被吵翻的问题。

## §3 · 三个裁决（重构）

### 裁决一（H2）：用量需求是不是被低估了？

- **论证链重排为三层**（修循环论证）：
  a. **独立证据**：官方 usage 锚——Google 480T→3.2Q/月、豆包 180T/日、Codex WAU 5 个月 8x、Claude Code 9 个月 $0→$2.5B（实测消耗，不含强度假设）；
  b. **机制解释**：任务谱系（解释消耗为何爆炸）；
  c. **形式化工具**：缺口模型（需求 3.94x vs 供给 2.85x → 0.70→0.97），**明示"需求线含强度假设，不作独立印证"**。
- **双口径限定**：结论是"**用量需求**被低估"；付费需求另算（中国 MaaS 实际变现 $0.1-0.2/MTok）。
- **敏感性必写**："若豆包口径含 cache 或中国线增速砍半，结论翻转——这是 H2 最脆弱的假设。"
- **强度**：有证据有争议。

### 裁决二（H3）：计量单位战争谁在赢？（瘦身 + 历史框架升级）

- **核心升级**：用"钟摆模型 + 两部制电价 + 做空成本曲线"替代民调数据（历史审查：证据强度更高）。
  - 钟摆：计量口径在"服务口径↔计量口径"间摆动，驱动变量是边际成本可预测性（电信 2017 无限套餐 → 2026 Airtel 悔意）。
  - 两部制电价（1897 起）：席位费 + 按量 = 百年答案；AI 的"E7 捆绑 + Copilot 底座按量"是数字重演。
  - **裁决公式（servitization 判据）：结果定价成立 ⟺ 单位任务成本下降斜率 > 竞争压价斜率**（Xerox click 成功六十年 / MoviePass 两年烧死）。
- **当前读数**：混合制暂赢（Futurum 21.7% vs 20.1% 持平点，标"偏好口径"）；Copilot 官方转按量是首个厂商承认任务强度差击破定价模型（引文须逐字核）。**AWU 的历史身份是 Xerox click 的重演——毛利引擎还是年金，取决于上面那条斜率公式。**
- 强度：有证据有争议。篇幅压缩，证据入图。

### 裁决三（H4）：前沿定价权能撑多久？（两组证据分列 + AOL 镜像）

- **需求证据 ≠ 定价权证据，分开摆**：
  - 需求强度：Claude Code $2.5B ARR、$1M+ 客户两月翻倍（空头也承认的硬牌）；
  - 定价权：GPT-5.5 翻倍、Fable $10/$50、K3=K2 5x、Windsurf 涨价（已实施且守住的提价）。
- **补贴的镜像（历史审查最强补充）**：**AOL 1996**——$19.95 包月取代计时，数周百万新用户、网络瘫痪、19 州总检察长介入。包月补贴=渗透武器+毛利毒药，是窗口不是稳态；且 AOL 有拨号存量续命，前沿实验室没有。
- **B 面（开源压缩）**：能力收敛（开源落后 ~4 个月、SWE-bench 差 14.9 分【修正 v1 的 8 分】）；K3 定价上探 $3/$15 = Sonnet 价——**放定价权一侧**（"开源=便宜"在前沿档失效），能力收敛才放压缩一侧。顶端护城河是长程可靠性（错误恢复 5 倍差）。
- **结论表述**："按量化已在发生（Copilot 6/1、Codex 4 月），问题只剩节奏"；自校验亮明："8x 增速与价格持平，明年必须让位一个。"
- 强度：假说待裁决。

## §4 · 边界：瓶颈接力（口径修正 + 回马枪 + 反方入列）

- **要读者相信**：任务通胀不豁免物理约束——它加重约束；瓶颈不会消失，只会迁移（偶尔回访）。
- **论证**：四棒迁移（GPU→HBM→CoWoS→电力）+ 三规律（envelope 上升、绕行决定刚性、租金周期化）+ 电力是"唯一资本无法**速成**的"（并网 5 年、变压器 36-48 月、燃机 2028 交付）。
- **口径声明（新增）**：缺口模型的 2.85x 是铭牌产能口径；电力延期兑现则为交付缺口，供给下修、缺口更紧。
- **回马枪**：棒会回访——HBM envelope 上升（Rubin 288GB→Rubin Ultra 1TB）。
- **反方入列（诚实条款）**：GE Vernova CEO 自称燃机"really not the gating item"（瓶颈在 EPC/许可/燃料）；"2030 售罄"是 CEO 预期非事实；MS 测算 2028 电力缺口 base case 基本收敛——写入 B 面。
- **PJM 更新**：容量缺口 6,625→**6,831MW**（2028/29 拍卖，连续第四次触顶）——方向强化。

## §5 · 副线：谁在为扩张付账（数字修正 + 历史硬拼 + 收入质量）

- **要读者相信**：2026 是内部现金工程转向杠杆工程的转折年；先断的不是需求，是融资。
- **修正**：Amazon FY2025 capex/OCF = **94%**（v1 的 113% 系 FY2024 OCF 错配；更硬的写法：FCF $38B→$11B）；合计口径改 "~90-100%（口径敏感）"。"Meta 订单缩水" → "**发债认购缩水 23%**（$125B→$96B，仍 3.8 倍覆盖）"。CoreWeave 评级更新 **BB+**（2026-05）。
- **历史硬拼（替换泛泛对照）**：Lucent/Nortel 2000 年对客户放款 $1.5B/$1.4B → 2001 亏损 $14.2B/$24.5B，并排 NVIDIA $100B→$30B、Broadcom-Apollo $35B XPV。
- **先例矩阵（空头审查）**：电信 2001（崩）/ 页岩（崩）/ Intel（裂）/ 公用事业（监管保底幸存）/ Amazon（从未到 100%，引证无效）——**样本里没有"无保底回报+100%+活下来"的先例**。
- **收入质量（新增）**：循环成分单列（厂商兜底 $6.3B、最大客户即股东、供应商融资），"净新增外部需求"与报表收入分离；折旧会计（延长年限美化 OCF）入监测。
- **"第一块砖已掉"**：Blue Owl 拒资 Oracle $10B、Oracle CDS 2009 来最高、微软单日 -$357B——比"松动信号"更准确的表述。
- 副线纪律：篇幅约主线一半；OpenAI 承诺细目全进下钻卡。

## §6 · 裁决、表达与分支开关（全新结构：可执行版）

**裁决（当前证据权重）**：H2 偏成立（用量口径）/ H3 混合制暂赢 / H4 真但靠补贴、必按量化。
**总结论**："换尺子是真的，量出来的数是肿的——但肿数字也是数字。按新尺子：**钱继续向上游和前沿流；模型层中端被挤死（产业链中游恰是租金所在）；链条先断的不是需求，是融资。**"（修正 v1"中间层被挤死"与 §4 的矛盾）

**中间层三分化（历史审查）**：套利旧尺子者归零 / 治理者新生（AI 版 FinOps：任务成本治理）/ 管道者被资本开支压垮。

**表达表（买方审查）**：每条裁决 → 可执行表达
| 裁决 | 表达 | 时钟 |
| --- | --- | --- |
| H2/瓶颈租金 | 多瓶颈租金层（存储/封装/电网设备/受限电力） | 季度-2 年 |
| 融资链 | 信用视角规避 neocloud 高息层与循环结构（天然多头对冲） | 周盯 |
| H3 | 相对价值：多按任务计价、空纯席位 token 成本通胀 SaaS | 2-4 季度 |

**H2×H4 分支矩阵**：双成立 / H2 对 H4 错 / H2 错 H4 对 / 双输，各情形下持什么；信用空头是整本多头的 CapEx beta 对冲。

**触发器 v2（三栏：多头阈值/空头阈值 + 频率 + 组合冲击等级 + 读数带日期）**：原 8 条修正（需求失速写死为"官方 usage 增速连续两季下台阶"；降级光互连轮动），新增：信用利差（neocloud HY 走阔 >150-200bp）、利率、S-1/审计披露、电力拍卖与限价、折旧政策、AWU 单价通缩、循环融资条款、**任务 token 强度拐点（空头第一证伪器）**、**结果定价商加 usage cap（混合制失败信号）**、提价窗口长度。CoreWeave 一月 -19% 标"半触发状态"。
