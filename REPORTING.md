# REPORTING.md — 互动研究报告写作规范

> 本文件只回答一个问题：**这份报告怎么写**。不记录任何研究内容——研究事实、模型读数、证据底座全部以 dossier（`D:\research\ai-industry-chain`）为唯一事实源。改研究改 dossier，改形式改本文件。

## 0. 单透镜原则（最重要）

整份报告只用一把"尺子"量所有问题。当前透镜：**任务计量**（从 token 到任务的计量切换，如何重写产业链的利润分配）。任何不能用这把尺子讲清楚的内容，不进报告。透镜更换需重写本文件，不允许同一份报告里出现两把尺子。

## 1. 形式：

（参考 XVC 报告的形式，不抄内容）：不抄part等表面形式

* 按可独立裁决的子问题切开。

## 2. 页面 = 判断句标题 → 论述 → 图 → 出处 → 下钻卡

* **判断句标题**：先给答案再给证据（"X 是否成立？——现有证据偏支持"）。事实与猜想分开标注（"猜想："前缀）。

* **论述**：每页 2-3 段充分文字（治"文字太少"的病），但不堆研究笔记；内部行号/版本号/过程叙述一律不出现。

* **判断强度三级**：已验证 / 有证据有争议 / 假说待裁决。标题强度永远不超过证据强度。

* **图**：每页一张主图，D3 动态，全部元素可点击下钻（下钻卡：数值、口径、带日期出处）。

* **出处**：只标四类——公司披露 / 券商研究 / 行业官方 / 自研模型；带日期。

## 3. 数据纪律

* 页面所有数字只能来自 `js/data.js`；`data.js` 是 dossier 读数的镜像，**不允许在 data.js 里新增 dossier 中没有的数字**。

* 工作流：研究变化 → 先改 dossier（canonical owner）→ 再同步 data.js → 报告自动一致。

* 查无实据的写"未披露"，不估算。

## 4. 表达规范

* 写作对象是聪明的外行：每个概念首次出现给一句人话解释；不用内部代号。

* 比喻克制：不用读者需要停下来理解的比喻；判断用直陈句。

* 金句可以要，但必须是结论而不是修辞。

* 反证不藏：每章给"什么证据会推翻本章判断"；B 面与 A 面同框呈现。

## 5. 报告元素清单（必须有）

* [ ] 封面：一句话总判断 + as-of 日期 + 关键数字带

* [ ] Part 进度条

* [ ] 每 Part：战场切片 + 判断句标题页 + 主图 + 出处 + 下钻卡

* [ ] 裁决章：各假说的证据权重 + 分支条件（什么情况走 B 面）

* [ ] 反证触发器面板（不做文字清单，做可交互开关面板：当前读数/阈值/触发后走向）

* [ ] 引用清单（按四类来源分组）

* [ ] 时效声明：锚点日期、下次复核窗口

## 6. 部署与协作

* GitHub Pages 自动发布：push 到 `main` 即上线（<https://alphamao1.github.io/ai-industry-chain-report/）。>

* 本地预览：`npm run dev`。

* 修改顺序：dossier（研究）→ data.js（镜像）→ 图表/文案（形式）→ push。
