## 一、核心需求（这份报告要回答什么）

**一句话**：AI 的计量单位正在从 token 换成"任务"——这次换尺子，会重写产业链上谁在赚钱、现在的扩张还能撑多久。

**读者为什么需要它**（需求错位）：现在所有公开讨论都建立在 token 口径上——"token 降价所以 AI 通缩"、"token 暴涨所以算力紧缺"，两个说法同时为真又互相矛盾，因为尺子本身错了。读者真正要回答的是"我的钱该押在产业链哪一环"，而这需要一把对的尺子。

**报告的交付物**：不是全景扫描，而是一个裁决——按新尺子量一遍产业链，给出"谁受益、谁受损、哪里会先断"的证据化判断，并给出推翻条件。

## 二、总叙事逻辑（一根链条，六个扣）

```
一个错位（钩子）→ 旧尺子失灵（价格两层）→ 新尺子（任务谱系）
```

每一扣只回答一个问题，回答完才进下一扣；每扣末尾一句话把读者交给下一扣。判断分三级标注：**已验证 / 有证据有争议 / 假说待裁决**——标题强度永远不超过证据强度。

## 三、章节结构与论证方式

### 引子 · 错位：价格跌了 24 倍，预算却烧穿了 10 倍

- **要读者相信**：现在流行口径里有个解不开的矛盾。
- **论证**：两个并排放的硬事实——GPT-4→GPT-5 旗舰标价 29 个月降 24 倍（官方）；Uber 四个月烧完全年 AI 预算、微软内部取消 Claude Code 许可（权威媒体）。不讲道理，只摆矛盾，结尾一句："矛盾不在事实里，在尺子里。"
- **图**：双轨迹小图（价格下行线 vs 企业支出上行线）。

### §1 · 旧尺子：token 价格的两层世界

- **要读者相信**："AI 在降价"只对了一半——走量层通缩、前沿层通胀，单层口径必然误判。
- **论证**：前沿档 V 形价格轨迹（GPT-5 $1.25 谷底→GPT-5.5 Pro $30、Fable $10/$50）+ 中国旗舰集体涨价（K3=K2 的 5 倍）+ DeepSeek 永久 75 折独守地板。三层证据都是官方定价页。
- **过渡**：为什么有人愿意为前沿付 30 倍价？因为他们买的不是 token。

### §2 · 新尺子：任务强度谱系

- **要读者相信**：token 是中间品（钢水），任务才是产品（灯泡）；任务之间有 10³→10⁹ 六个数量级的强度差，而行业第一次有了官方"汇率"。
- **论证**：任务强度谱系逐档锚定（对话<1K、单文档 5K、多文档 10 万、agentic coding 10⁶-10⁷【斯坦福论文：1,200-3,500×】、应用构建 1 亿-40 亿）+ Salesforce 官方汇率 7,526 token/AWU。这是全报告的地基图。
- **过渡**：用这把尺子重新量三个被吵翻的问题。

### §3 · 三个裁决

**裁决一（H2）：需求是不是被低估了？**
- 论证：任务通胀（单任务 token 百倍千倍涨）× 供需缺口模型（需求 3.94x vs 供给 2.85x → 缺口 0.70→0.97）。任务级微观证据与总量模型互相印证。
- 强度：有证据有争议。反证：agent 闭环成功率停滞。

**裁决二（H3）：计量单位战争谁在赢？**
- 论证：三层结构收敛（底层按量：Copilot 官方"同价不可持续"转按量；中层 work unit：AWU/Intercom $0.99 按解决；顶层席位打包：E7 $99 捆绑）。当前赢家是"混合制"，不是纯结果制；Futurum 21.7% vs 20.1% 持平点。
- 强度：有证据有争议。反证：AWU 被证明是虚荣指标、无第二家大厂跟进。

**裁决三（H4）：前沿定价权能撑住吗？**
- 论证（这是全报告最锋利的一节，必须保留张力）：付费意愿极强（Claude Code $2.5B ARR、$1M+ 客户两月翻倍）但**定价权部分是补贴出来的**（SemiAnalysis：$200 订阅交付 $8-14k API 等效，利用率 6-11% 即负毛利）→ 必然走向按量化；开源压缩中端（SWE-bench 差 8 分、K3 定到 Sonnet 价）但顶端护城河是长程可靠性（错误恢复 5 倍差）。
- 强度：假说待裁决。自校验警示保留："8x 增速与价格持平不可兼得"。

### §4 · 边界：瓶颈接力（穿插章）

- **要读者相信**：任务经济学不豁免物理约束——任务越通胀，瓶颈咬得越紧；瓶颈不会消失，只会迁移。
- **论证**：四棒接力（GPU→HBM→CoWoS→电力）+ 三条规律（envelope 上升、绕行决定刚性、租金周期化）+ 第四棒电力是唯一资本无法加速的（并网 5 年、变压器 3-4 年、燃机 2028 售罄、PJM 首次容量缺口）。
- **位置设计**：这章同时是 H2 的边界条件——需求是真的，所以约束才更硬。过渡：需求真、瓶颈硬，那谁在掏钱填这个缺口？

### §5 · 副线：谁在为扩张付账（资产负债表）

- **要读者相信**：2026 是内部现金工程转向杠杆工程的转折年；链条最脆处不在超大厂，在 OpenAI 回路和 neocloud 高息层；顶部信号在债券市场，不在订单簿。
- **论证**：CapEx/OCF 逼近 100%（10 年均值 40%）→ 融资三层解剖（发债 $159B YTD / 表外 SPV $27B / 供应商循环 NVIDIA $100B 作废转 $30B、Broadcom-Apollo $35B）→ 2001 电信对照（同构：同步性；差异：本轮有真实收入）→ 首批松动信号（Meta 订单缩水 23%、Oracle 债被拒购、要求谷歌背书）。
- 强度：已验证（事实）+ 有争议（含义）。这章是副线：篇幅约主线一半，不抢主轴。

### §6 · 裁决与分支开关

- 总结三个假说的当前证据权重（H2 偏成立 / H3 混合制暂赢 / H4 真但有裂缝），给出投资含义：**按新尺子，钱继续向上游和前沿流，中间层被挤死；链条先断的地方不是需求，是融资。**

## 四、自检（逻辑严密性与"内外分离"审查）

**逻辑风险点（写作时必须处理，不能藏）**：
1. **H2 与缺口模型的循环性**：缺口模型的需求增速本身就含任务强度假设——不能用它"证明"任务通胀。处理：微观任务证据（谱系）与总量模型呈现为**互相印证**，不说"证明"。
2. **H4 与价格层自校验的张力**：定价权成立与"$226B→$1,366B 不可能"看似矛盾。处理：不调和，直接作为裁决点——这恰恰说明"增速或价格必有一个让位"。
3. **瓶颈章的位置**：放在裁决后是边界条件，放在裁决前会打断主线。选定放 §4，并显式写出"这章同时是 H2 的边界"。
4. **副线不抢戏**：资产负债表只回答"谁先断"，不展开成第二份报告；OpenAI 承诺细目放下钻卡不进正文。
5. **K3 只作价格/开源证据**（暴跌关联已按证据抛弃），不再提及市场归因。

**内外分离清单（正文与下钻卡一律不出现）**：RPP-IN/SIG/ASM 行号、模型版本号（v0.3.1 等）、lifecycle/canonical/dossier/legacy 术语、文件路径、"七路调研"等过程叙述。来源只标四类：公司披露 / 券商研究 / 行业官方 / 自研模型。

**证据纪律**：每章核心数字都能在数据层找到出处；查无实据的（K3 训练成本、V4 训练成本）写"未披露"而不估算。
